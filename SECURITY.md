# Security Policy

## Supported Versions
Security updates are applied only to the latest stable release of UCP.

## Reporting a Vulnerability

If you discover a vulnerability in UCP, please report it by emailing **security@ucp.dev**. Do not open a public GitHub issue.

Please include:
* The specific crate affected (`ucp-core`, `ucp-synthesizer`, etc.).
* A description of the vulnerability.
* Steps to reproduce the vulnerability.
* If known, the potential impact.

We will acknowledge receipt within 48 hours and aim to provide a resolution or timeline within 7 days.

## Security Architecture & Constraints

Because UCP's core value proposition involves analyzing third-party source code via AI, we enforce strict security boundaries at the architectural level. All contributors and subagent workflows must adhere to the following constraints:

### 1. Local-First AI Inference
By design, the `ucp-synthesizer` pipeline **MUST NOT** make network requests to external AI APIs (such as OpenAI or Anthropic) by default. It exclusively targets a local Ollama instance (`http://localhost:11434`). 

If a user explicitly overrides this via the `--llm-provider openai` flag, the CLI **MUST** emit a `SECURITY WARNING` to stderr:
```text
WARNING: Sending extracted third-party source code to an external API endpoint.
```

### 2. Source Code Sandboxing
The Extraction Agent (`ucp-synthesizer/src/extract/`) is strictly sandboxed. When parsing cloned repositories, it is forbidden from reading:
* Environment files (`.env`, `.env.local`)
* Cryptographic keys or certificates (`*.pem`, `*.key`, `*.p12`)
* CI/CD configuration (`.github/`, `.circleci/`)
* Test fixtures (`tests/`)
* Build artifacts (`target/`, `dist/`, `node_modules/`)

It is restricted exclusively to:
* `**/src/**/*.rs`
* `**/src/**/*.tsx`
* `**/components/**`

### 3. SPDX License Gating
The Discovery Agent (`ucp-synthesizer/src/discovery.rs`) automatically rejects any repository that does not declare an SPDX-permissive license (`MIT`, `Apache-2.0`, `BSD-2-Clause`, `BSD-3-Clause`, `ISC`, `0BSD`, `Unicode-DFS-2016`). Copyleft licenses (`GPL`, `AGPL`) are hard-failures. The pipeline will halt parsing a repository if its license cannot be parsed or is non-compliant.

### 4. Static HTML Curation UI
The conflict resolution output generated by the CLI (`ucp resolve --format html`) is a static HTML file containing source code snippets from external repositories. This HTML file is designed to run entirely in the browser with zero backend dependencies. This ensures raw source code from discovered repositories never leaves the user's local machine during human review.

### 5. Dependency Auditing
All dependencies added to `Cargo.toml` files must be audited for supply-chain risks. The use of `cargo-audit` is enforced in CI. Dependencies with known critical or high vulnerabilities will block the CI pipeline until resolved or replaced.
